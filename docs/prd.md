# เอกสารความต้องการ

## 1. ภาพรวมแอปพลิเคชัน

### 1.1 ชื่อแอปพลิเคชัน
ระบบจัดคิวหมอสำหรับ GTA V RP Server

### 1.2 คำอธิบาย
เว็บแอปพลิเคชันสำหรับจัดการคิวหมอในเกม GTA V (FiveM/GTARP) โดยดึงข้อมูลจาก game server เพื่อแสดงรายชื่อหมอที่ออนไลน์ และจัดการสถานะการรับคิวของหมอแต่ละคน รองรับการใช้งานทั้งบน desktop และ mobile

## 2. ผู้ใช้งานและสถานการณ์การใช้งาน

### 2.1 ผู้ใช้งานหลัก
- **คนรัน OP (Operator)**: ผู้ดูแลระบบคิว มีหน้าที่เลื่อนคิวและจัดการระบบ
- **หมอในเกม**: ผู้เล่นที่รับบทเป็นหมอ สามารถเปลี่ยนสถานะตัวเองได้
- **แอดมิน**: ผู้จัดการระบบ สามารถเพิ่ม/ลบชื่อหมอในโหมด Manual

### 2.2 สถานการณ์การใช้งานหลัก
- Operator ต้องการเลื่อนคิวไปยังหมอคนถัดไปที่พร้อมรับงาน
- หมอต้องการเปลี่ยนสถานะเมื่อไปทำกิจกรรม หรือพักชั่วคราว
- ระบบต้องแสดงรายชื่อหมอที่ออนไลน์ในเกมแบบ real-time
- แอดมินต้องการจัดการรายชื่อหมอเมื่อไม่สามารถดึงข้อมูลจาก server ได้

## 3. โครงสร้างหน้าและฟังก์ชัน

### 3.1 โครงสร้างหน้า
```
├── หน้าหลัก (Queue Management)
│   ├── ส่วนแสดงคิว OP
│   ├── ส่วนกิจกรรม
│   ├── ส่วนเหม่อ
│   ├── ส่วนออกเวร
│   └── ส่วนไป Story
└── หน้าตั้งค่า (Settings)
    ├── ตั้งค่า Server URL
    └── โหมด Manual (เพิ่ม/ลบชื่อหมอ)
```

### 3.2 หน้าหลัก (Queue Management)

#### 3.2.1 ส่วนแสดงคิว OP
- แสดงชื่อ Operator ที่ด้านบนสุด
- แสดงรายชื่อหมอที่พร้อมรับคิว โดยมีไอคอน 🟢 นำหน้าชื่อ
- แสดงไอคอน 👉 ทั้งซ้ายและขวาของชื่อหมอที่จะรับคิวถัดไป
- ปุ่ม「ถัดไป」สำหรับเลื่อนตัวชี้ไปยังคนถัดไป (วนจากบนลงล่าง แล้ววนกลับขึ้นบน)
- เมนูสำหรับแต่ละคน: ไปกิจกรรม, เหม่อ, ออกเวร, ไป Story, ลง OP

#### 3.2.2 ส่วนกิจกรรม
- แสดงรายชื่อหมอที่ไปทำกิจกรรม
- เมนูสำหรับแต่ละคน: ขึ้น OP, เหม่อ, ออกเวร, ไป Story

#### 3.2.3 ส่วนเหม่อ
- แสดงรายชื่อหมอที่ไม่ตอบสนอง/AFK
- เมนูสำหรับแต่ละคน: ขึ้น OP, ไปกิจกรรม, ออกเวร, ไป Story

#### 3.2.4 ส่วนออกเวร
- แสดงรายชื่อหมอที่ออกจากงานแล้ว
- เมนูสำหรับแต่ละคน: ขึ้น OP, ไปกิจกรรม, เหม่อ, ไป Story

#### 3.2.5 ส่วนไป Story
- แสดงรายชื่อหมอที่ไปทำ Story Mode
- เมนูสำหรับแต่ละคน: ขึ้น OP, ไปกิจกรรม, เหม่อ, ออกเวร

### 3.3 หน้าตั้งค่า (Settings)

#### 3.3.1 ตั้งค่า Server URL
- ช่องกรอก Server IP/URL สำหรับเชื่อมต่อ FiveM server
- ปุ่ม「บันทึก」เพื่อบันทึกการตั้งค่า
- ปุ่ม「ทดสอบการเชื่อมต่อ」เพื่อตรวจสอบว่าเชื่อมต่อได้หรือไม่

#### 3.3.2 โหมด Manual
- ปุ่ม「เปิดโหมด Manual」เมื่อไม่สามารถดึงข้อมูลจาก server ได้
- ช่องกรอกชื่อหมอ
- ปุ่ม「เพิ่มหมอ」เพื่อเพิ่มชื่อเข้าระบบ
- ปุ่ม「ลบ」ข้างชื่อแต่ละคนเพื่อลบออกจากระบบ

## 4. กฎทางธุรกิจและตรรกะ

### 4.1 การจัดการคิว
- Operator มีได้เพียงคนเดียวในระบบ
- ตัวชี้คิว (👉) เลื่อนจากบนลงล่างตามลำดับ เมื่อถึงคนสุดท้ายจะวนกลับไปที่คนแรก
- เมื่อหมอเปลี่ยนสถานะ ตัวชี้จะยังคงอยู่ที่ตำแหน่งเดิม (ไม่เลื่อนตาม)

### 4.2 การเปลี่ยนสถานะ
- หมอสามารถอยู่ในสถานะใดสถานะหนึ่งเท่านั้น ไม่สามารถอยู่หลายสถานะพร้อมกันได้
- เมื่อเปลี่ยนสถานะ ชื่อจะถูกย้ายไปยังส่วนที่เกี่ยวข้องทันที
- การกด「ขึ้น OP」จะย้ายหมอกลับไปยังส่วนคิว OP และเข้าคิวท้ายสุด

### 4.3 การดึงข้อมูลจาก Server
- ระบบดึงข้อมูลรายชื่อผู้เล่นจาก FiveM server API ตาม URL ที่ตั้งค่าไว้
- แสดงเฉพาะหมอที่มีสถานะ online ในเกม
- อัปเดตข้อมูลทุก 30 วินาที หรือเมื่อมีการเปลี่ยนแปลง
- หากดึงข้อมูลไม่สำเร็จ ให้แสดงข้อความแจ้งเตือนและเปลี่ยนเป็นโหมด Manual

### 4.4 การจัดเก็บข้อมูล
- ข้อมูลสถานะของหมอแต่ละคนจัดเก็บใน Supabase database
- ข้อมูลคงอยู่แม้ refresh หน้า
- บันทึกประวัติการเปลี่ยนสถานะพร้อมเวลา

## 5. สถานการณ์ข้อยกเว้นและกรณีขอบเขต

| สถานการณ์ | การจัดการ |
|-----------|----------|
| ไม่สามารถเชื่อมต่อ FiveM server | แสดงข้อความแจ้งเตือน และเปิดโหมด Manual อัตโนมัติ |
| หมอทุกคนออกจากคิว OP | ตัวชี้หายไป ปุ่ม「ถัดไป」ไม่สามารถกดได้ |
| มีหมอเพียงคนเดียวในคิว | ตัวชี้ชี้ที่คนเดียว กดปุ่ม「ถัดไป」แล้วยังคงชี้ที่คนเดิม |
| Server URL ไม่ถูกต้อง | แสดงข้อความ「ไม่สามารถเชื่อมต่อได้ กรุณาตรวจสอบ URL」|
| เพิ่มชื่อซ้ำในโหมด Manual | แสดงข้อความ「ชื่อนี้มีอยู่ในระบบแล้ว」|
| ลบหมอที่กำลังถูกชี้โดยตัวชี้ | ตัวชี้เลื่อนไปยังคนถัดไปอัตโนมัติ |

## 6. เกณฑ์การตรวจรับ

1. เข้าหน้าหลัก แสดงส่วนคิว OP พร้อมรายชื่อหมอที่ออนไลน์ (หรือรายชื่อที่เพิ่มในโหมด Manual)
2. กดปุ่ม「ถัดไป」ตัวชี้ 👉 เลื่อนไปยังชื่อถัดไปตามลำดับ และวนกลับเมื่อถึงคนสุดท้าย
3. เลือกหมอคนหนึ่ง กดเมนู「ไปกิจกรรม」ชื่อหมอย้ายไปส่วนกิจกรรมทันที
4. ในส่วนกิจกรรม กดเมนู「ขึ้น OP」ชื่อหมอกลับไปยังส่วนคิว OP ที่ตำแหน่งท้ายสุด
5. เข้าหน้าตั้งค่า กรอก Server URL กดปุ่ม「บันทึก」และ「ทดสอบการเชื่อมต่อ」แสดงผลการเชื่อมต่อ
6. เปิดโหมด Manual เพิ่มชื่อหมอใหม่ ชื่อปรากฏในส่วนคิว OP
7. Refresh หน้า ข้อมูลสถานะและตำแหน่งคิวยังคงเหมือนเดิม

## 7. ฟังก์ชันที่ไม่ดำเนินการในเวอร์ชันนี้

- การแจ้งเตือนผ่าน Discord Bot
- ระบบสิทธิ์การเข้าถึงแบบหลายระดับ (Admin/Moderator/User)
- ประวัติการรับคิวของหมอแต่ละคน
- สถิติการทำงานของหมอ (จำนวนคิวที่รับ, เวลาเฉลี่ย)
- การส่งข้อความหรือแจ้งเตือนถึงหมอในเกม
- ระบบจองคิวล่วงหน้า
- การกำหนดลำดับความสำคัญของคิว (Priority Queue)
- การบันทึกเสียงหรือวิดีโอการทำงาน
- การส่งออกรายงานเป็นไฟล์ Excel/PDF
- Dark/Light mode toggle
- การแปลภาษาอัตโนมัติ
- การซิงค์ข้อมูลกับระบบอื่นๆ นอกเหนือจาก FiveM server
