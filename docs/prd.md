# เอกสารความต้องการ

## 1. ภาพรวมแอปพลิเคชัน

### 1.1 ชื่อแอปพลิเคชัน
ระบบจัดคิวหมอสำหรับ GTA V RP Server

### 1.2 คำอธิบาย
เว็บแอปพลิเคชันสำหรับจัดการคิวหมอในเกม GTA V (FiveM/GTARP) โดยดึงข้อมูลจาก game server เพื่อแสดงรายชื่อหมอที่ออนไลน์ และจัดการสถานะการรับคิวของหมอแต่ละคน พร้อมระบบยศ ระบบบัญชีผู้ใช้ ระบบนับเวลาทำงาน และระบบใบเตือน รองรับการใช้งานทั้งบน desktop และ mobile

## 2. ผู้ใช้งานและสถานการณ์การใช้งาน

### 2.1 ผู้ใช้งานหลัก
- **Admin**: ผู้ดูแลระบบระดับสูงสุด มีสิทธิ์ทุกอย่าง
- **ผอ (ผู้อำนวยการ)**: ยศพิเศษระดับสูง มีสิทธิ์ใกล้เคียง admin
- **คนรัน OP (Operator)**: ผู้ดูแลระบบคิว มีหน้าที่เลื่อนคิวและจัดการระบบ
- **หมอในเกม**: ผู้เล่นที่รับบทเป็นหมอ สามารถเปลี่ยนสถานะตัวเองได้
- **ผู้ใช้งานทั่วไป**: ผู้ที่มียศอื่นๆ ตามที่ admin กำหนด

### 2.2 สถานการณ์การใช้งานหลัก
- Admin สร้างบัญชีให้หมอใหม่ พร้อมกำหนดยศและสิทธิ์
- หมอ login เข้าระบบ ระบบเริ่มนับเวลาทำงานอัตโนมัติ
- หมอเปลี่ยนสถานะตัวเอง (ไปกิจกรรม, เหม่อ, ออกเวร, ไป Story)
- Operator เลื่อนคิวไปยังหมอคนถัดไปที่พร้อมรับงาน
- หมอดู dashboard ส่วนตัวเพื่อตรวจสอบชั่วโมงทำงานของตัวเอง
- ผอ/Admin ดู dashboard ภาพรวมเพื่อตรวจสอบชั่วโมงทำงานของทุกคน
- ผอ/Admin ออกใบเตือนให้หมอที่ทำงานไม่ถึงเวลาที่กำหนด
- Admin จัดการยศและสิทธิ์ของผู้ใช้งาน

## 3. โครงสร้างหน้าและฟังก์ชัน

### 3.1 โครงสร้างหน้า
```
├── หน้า Login
├── หน้าหลัก (Queue Management)
│   ├── ส่วนแสดงคิว OP
│   ├── ส่วนกิจกรรม
│   ├── ส่วนเหม่อ
│   ├── ส่วนออกเวร
│   └── ส่วนไป Story
├── หน้า Dashboard ส่วนตัว
├── หน้า Dashboard ภาพรวม (Admin/ผอ)
├── หน้าจัดการบัญชี (Account Management)
├── หน้าจัดการยศ (Role Management)
├── หน้าใบเตือน (Warning System)
└── หน้าตั้งค่า (Settings)
    ├── ตั้งค่า Server URL
    ├── ตั้งค่าชั่วโมงทำงาน
    └── โหมด Manual
```

### 3.2 หน้า Login

#### 3.2.1 ฟังก์ชัน
- ช่องกรอกชื่อผู้ใช้
- ช่องกรอกรหัสผ่าน
- ปุ่ม「เข้าสู่ระบบ」
- เมื่อ login สำเร็จ ระบบเริ่มนับเวลาทำงานทันที

### 3.3 หน้าหลัก (Queue Management)

#### 3.3.1 ส่วนแสดงคิู OP
- แสดงชื่อ Operator ที่ด้านบนสุด
- แสดงรายชื่อหมอที่พร้อมรับคิว โดยมีไอคอน 🟢 นำหน้าชื่อ
- แสดงไอคอน 👉 ทั้งซ้ายและขวาของชื่อหมอที่จะรับคิวถัดไป
- ปุ่ม「ถัดไป」สำหรับเลื่อนตัวชี้ไปยังคนถัดไป (วนจากบนลงล่าง แล้ววนกลับขึ้นบน)
- แต่ละคนมีเมนูสำหรับเปลี่ยนสถานะ: ไปกิจกรรม, เหม่อ, ออกเวร, ไป Story
- แต่ละคนกดเมนูได้เฉพาะของตัวเอง หรือผู้ที่มีสิทธิ์กดเมนูของคนอื่น

#### 3.3.2 ส่วนกิจกรรม
- แสดงรายชื่อหมอที่ไปทำกิจกรรม
- เมนูสำหรับแต่ละคน: ขึ้น OP, เหม่อ, ออกเวร, ไป Story

#### 3.3.3 ส่วนเหม่อ
- แสดงรายชื่อหมอที่ไม่ตอบสนอง/AFK
- เมนูสำหรับแต่ละคน: ขึ้น OP, ไปกิจกรรม, ออกเวร, ไป Story

#### 3.3.4 ส่วนออกเวร
- แสดงรายชื่อหมอที่ออกจากงานแล้ว
- เมนูสำหรับแต่ละคน: ขึ้น OP, ไปกิจกรรม, เหม่อ, ไป Story

#### 3.3.5 ส่วนไป Story
- แสดงรายชื่อหมอที่ไปทำ Story Mode
- เมนูสำหรับแต่ละคน: ขึ้น OP, ไปกิจกรรม, เหม่อ, ออกเวร

### 3.4 หน้า Dashboard ส่วนตัว

#### 3.4.1 ฟังก์ชัน
- แสดงชั่วโมงทำงานของตัวเองในวันนี้
- แสดงชั่วโมงทำงานของตัวเองในอาทิตย์นี้
- แสดงประวัติการ login/logout ย้อนหลัง
- แสดงประวัติการเปลี่ยนสถานะย้อนหลัง

### 3.5 หน้า Dashboard ภาพรวม (Admin/ผอ)

#### 3.5.1 ฟังก์ชัน
- แสดงรายชื่อทุกคนพร้อมชั่วโมงทำงาน
- Filter ตามวันที่: วันเดียว / ช่วงเวลา / อาทิตย์
- Filter ตามสถานะการทำงาน:
  + คนที่ทำงานไม่ถึงเวลาที่กำหนด
  + คนที่ทำงานถึงเวลาที่กำหนด
  + คนที่ทำงานเกินเวลาที่กำหนด
- สรุปชั่วโมงทำงานรายคน รายอาทิตย์
- ปุ่ม「ออกใบเตือน」สำหรับแต่ละคน

### 3.6 หน้าจัดการบัญชี (Account Management)

#### 3.6.1 ฟังก์ชัน
- แสดงรายชื่อบัญชีทั้งหมด
- ปุ่ม「สร้างบัญชีใหม่」(สำหรับผู้มีสิทธิ์)
  + ช่องกรอกชื่อผู้ใช้
  + ช่องกรอกรหัสผ่านเริ่มต้น
  + เลือกยศ
  + ปุ่ม「บันทึก」
- แต่ละบัญชีมีปุ่ม「แก้ไข」และ「ลบ」(สำหรับผู้มีสิทธิ์)
- ผู้ใช้สามารถแก้ไขชื่อและเปลี่ยนรหัสผ่านของตัวเองได้

### 3.7 หน้าจัดการยศ (Role Management)

#### 3.7.1 ฟังก์ชัน
- แสดงรายการยศทั้งหมด (admin และ ผอ เป็นยศพิเศษที่มีอยู่เสมอ)
- ปุ่ม「สร้างยศใหม่」(สำหรับ admin)
  + ช่องกรอกชื่อยศ
  + เลือกสิทธิ์ต่างๆ:
    - สร้างบัญชีให้คนอื่น
    - จัดการยศ (สร้าง/แก้ไข/ลบยศ)
    - กดเมนูสถานะของคนอื่น
    - ดู dashboard ภาพรวม
    - ออกใบเตือน
    - เข้าหน้าตั้งค่าระบบ
    - เพิ่ม/ลบหมอในระบบ
  + ปุ่ม「บันทึก」
- แต่ละยศมีปุ่ม「แก้ไข」และ「ลบ」(ยกเว้น admin และ ผอ)

### 3.8 หน้าใบเตือน (Warning System)

#### 3.8.1 ฟังก์ชัน
- แสดงรายการใบเตือนทั้งหมด (สำหรับ admin/ผอ)
- ปุ่ม「ออกใบเตือนใหม่」(สำหรับ admin/ผอ)
  + เลือกผู้รับใบเตือน
  + ช่องกรอกเหตุผล
  + ปุ่ม「บันทึก」
- แสดงประวัติใบเตือนที่ตัวเองรับ (สำหรับผู้ใช้ทั่วไป)
- แสดงประวัติใบเตือนที่ตัวเองออก (สำหรับ admin/ผอ)

### 3.9 หน้าตั้งค่า (Settings)

#### 3.9.1 ตั้งค่า Server URL
- ช่องกรอก Server IP/URL สำหรับเชื่อมต่อ FiveM server
- ปุ่ม「บันทึก」เพื่อบันทึกการตั้งค่า
- ปุ่ม「ทดสอบการเชื่อมต่อ」เพื่อตรวจสอบว่าเชื่อมต่อได้หรือไม่

#### 3.9.2 ตั้งค่าชั่วโมงทำงาน
- ช่องกรอกจำนวนชั่วโมงทำงานต่ออาทิตย์ที่กำหนด
- ปุ่ม「บันทึก」

#### 3.9.3 โหมด Manual
- ปุ่ม「เปิดโหมด Manual」เมื่อไม่สามารถดึงข้อมูลจาก server ได้
- ช่องกรอกชื่อหมอ
- ปุ่ม「เพิ่มหมอ」เพื่อเพิ่มชื่อเข้าระบบ
- ปุ่ม「ลบ」ข้างชื่อแต่ละคนเพื่อลบออกจากระบบ

## 4. กฎทางธุรกิจและตรรกะ

### 4.1 ระบบยศและสิทธิ์
- ยศพิเศษ admin และ ผอ มีอยู่เสมอในระบบ ไม่สามารถลบได้
- admin มีสิทธิ์ทุกอย่างในระบบ
- ผอ มีสิทธิ์ใกล้เคียง admin
- ยศอื่นๆ สามารถสร้างและกำหนดสิทธิ์ได้ตามต้องการ
- แต่ละฟังก์ชันในระบบสามารถกำหนดเป็นสิทธิ์ในการสร้าง/แก้ไขยศได้

### 4.2 ระบบบัญชีผู้ใช้
- admin หรือผู้มีสิทธิ์สร้างบัญชี สามารถสร้างบัญชีใหม่ได้ โดยกำหนดชื่อผู้ใช้ รหัสผ่านเริ่มต้น และยศ
- ผู้ใช้สามารถเปลี่ยนชื่อตัวเองและเปลี่ยนรหัสผ่านได้ (เฉพาะของตัวเองเท่านั้น)
- ผู้ใช้ไม่สามารถเปลี่ยนยศของตัวเองได้

### 4.3 ระบบสถานะและห้อง
- แต่ละคนสามารถอยู่ในห้องใดห้องหนึ่งเท่านั้น:
  + ห้อง OP (คิวปกติ พร้อมรับงาน)
  + ห้องกิจกรรม
  + ห้องเหม่อ (AFK)
  + ห้องออกเวร
  + ห้องไป Story
- แต่ละคนกดเมนูเปลี่ยนสถานะได้เฉพาะของตัวเอง
- ผู้ที่มีสิทธิ์「กดเมนูสถานะของคนอื่น」สามารถเปลี่ยนสถานะให้คนอื่นได้
- เมื่อเปลี่ยนสถานะ ชื่อจะถูกย้ายไปยังห้องที่เกี่ยวข้องทันที

### 4.4 การจัดการคิว
- Operator มีได้เพียงคนเดียวในระบบ
- ตัวชี้คิว (👉) เลื่อนจากบนลงล่างตามลำดับ เมื่อถึงคนสุดท้ายจะวนกลับไปที่คนแรก
- เมื่อหมอเปลี่ยนสถานะ ตัวชี้จะยังคงอยู่ที่ตำแหน่งเดิม (ไม่เลื่อนตาม)
- การกด「ขึ้น OP」จะย้ายหมอกลับไปยังส่วนคิว OP และเข้าคิวท้ายสุด

### 4.5 ระบบนับเวลาทำงาน
- เริ่มนับเวลาทำงานทันทีที่ผู้ใช้ login เข้าระบบ
- หยุดนับเวลาเมื่อผู้ใช้ logout ออกจากระบบ
- บันทึก session ทุกครั้งที่ login/logout พร้อม timestamp
- คำนวณชั่วโมงทำงานรวมในแต่ละวัน แต่ละอาทิตย์

### 4.6 ระบบใบเตือน
- admin และ ผอ สามารถออกใบเตือนให้ผู้ใช้คนใดก็ได้
- ใบเตือนต้องระบุเหตุผล
- บันทึกประวัติใบเตือนพร้อม timestamp ผู้ออก และผู้รับ

### 4.7 การดึงข้อมูลจาก Server
- ระบบดึงข้อมูลรายชื่อผู้เล่นจาก FiveM server API ตาม URL ที่ตั้งค่าไว้
- แสดงเฉพาะหมอที่มีสถานะ online ในเกม
- อัปเดตข้อมูลทุก 30 วินาที หรือเมื่อมีการเปลี่ยนแปลง
- หากดึงข้อมูลไม่สำเร็จ ให้แสดงข้อความแจ้งเตือนและเปลี่ยนเป็นโหมด Manual

### 4.8 การจัดเก็บข้อมูล
- ข้อมูลบัญชีผู้ใช้ ยศ สิทธิ์ จัดเก็บใน Supabase database
- ข้อมูลสถานะของหมอแต่ละคนจัดเก็บใน Supabase database
- ข้อมูล session การทำงาน (login/logout timestamp) จัดเก็บใน Supabase database
- ข้อมูลใบเตือนจัดเก็บใน Supabase database
- ข้อมูลคงอยู่แม้ refresh หน้า

## 5. สถานการณ์ข้อยกเว้นและกรณีขอบเขต

| สถานการณ์ | การจัดการ |
|-----------|----------|
| ไม่สามารถเชื่อมต่อ FiveM server | แสดงข้อความแจ้งเตือน และเปิดโหมด Manual อัตโนมัติ |
| หมอทุกคนออกจากคิว OP | ตัวชี้หายไป ปุ่ม「ถัดไป」ไม่สามารถกดได้ |
| มีหมอเพียงคนเดียวในคิว | ตัวชี้ชี้ที่คนเดียว กดปุ่ม「ถัดไป」แล้วยังคงชี้ที่คนเดิม |
| Server URL ไม่ถูกต้อง | แสดงข้อความ「ไม่สามารถเชื่อมต่อได้ กรุณาตรวจสอบ URL」|
| เพิ่มชื่อซ้ำในโหมด Manual | แสดงข้อความ「ชื่อนี้มีอยู่ในระบบแล้ว」|
| ลบหมอที่กำลังถูกชี้โดยตัวชี้ | ตัวชี้เลื่อนไปยังคนถัดไปอัตโนมัติ |
| สร้างบัญชีซ้ำ | แสดงข้อความ「ชื่อผู้ใช้นี้มีอยู่ในระบบแล้ว」|
| ลบยศที่มีผู้ใช้งานอยู่ | แสดงข้อความ「ไม่สามารถลบยศที่มีผู้ใช้งานอยู่ได้」|
| ผู้ใช้พยายามแก้ไขข้อมูลคนอื่น | แสดงข้อความ「คุณไม่มีสิทธิ์ทำรายการนี้」|
| ผู้ใช้ logout แล้วลืม login กลับ | ระบบไม่นับเวลาทำงานในช่วงที่ logout |
| ผู้ใช้ปิดหน้าต่างโดยไม่ logout | ระบบตรวจจับและบันทึก logout อัตโนมัติ |

## 6. เกณฑ์การตรวจรับ

1. เข้าหน้า Login กรอกชื่อผู้ใช้และรหัสผ่าน กดปุ่ม「เข้าสู่ระบบ」เข้าสู่หน้าหลักสำเร็จ
2. เข้าหน้าหลัก แสดงส่วนคิว OP พร้อมรายชื่อหมอที่ออนไลน์ กดปุ่ม「ถัดไป」ตัวชี้ 👉 เลื่อนไปยังชื่อถัดไปตามลำดับ
3. เลือกหมอคนหนึ่ง กดเมนู「ไปกิจกรรม」ชื่อหมอย้ายไปส่วนกิจกรรมทันที
4. ในส่วนกิจกรรม กดเมนู「ขึ้น OP」ชื่อหมอกลับไปยังส่วนคิว OP ที่ตำแหน่งท้ายสุด
5. เข้าหน้า Dashboard ส่วนตัว แสดงชั่วโมงทำงานของตัวเองในวันนี้และอาทิตย์นี้
6. เข้าหน้า Dashboard ภาพรวม (สำหรับ admin/ผอ) แสดงรายชื่อทุกคนพร้อมชั่วโมงทำงาน Filter ตามวันที่และสถานะการทำงานได้
7. เข้าหน้าจัดการบัญชี กดปุ่ม「สร้างบัญชีใหม่」กรอกข้อมูล เลือกยศ กดปุ่ม「บันทึก」บัญชีใหม่ปรากฏในรายการ
8. เข้าหน้าจัดการยศ กดปุ่ม「สร้างยศใหม่」กรอกชื่อยศ เลือกสิทธิ์ กดปุ่ม「บันทึก」ยศใหม่ปรากฏในรายการ
9. เข้าหน้าใบเตือน กดปุ่ม「ออกใบเตือนใหม่」เลือกผู้รับ กรอกเหตุผล กดปุ่ม「บันทึก」ใบเตือนปรากฏในประวัติ
10. เข้าหน้าตั้งค่า กรอก Server URL กดปุ่ม「บันทึก」และ「ทดสอบการเชื่อมต่อ」แสดงผลการเชื่อมต่อ
11. กดปุ่ม Logout ออกจากระบบ ระบบหยุดนับเวลาทำงาน
12. Refresh หน้า ข้อมูลสถานะและตำแหน่งคิวยังคงเหมือนเดิม

## 7. ฟังก์ชันที่ไม่ดำเนินการในเวอร์ชันนี้

- การแจ้งเตือนผ่าน Discord Bot
- ประวัติการรับคิวของหมอแต่ละคน (จำนวนคิวที่รับ)
- สถิติการทำงานของหมอ (เวลาเฉลี่ยต่อคิว)
- การส่งข้อความหรือแจ้งเตือนถึงหมอในเกม
- ระบบจองคิวล่วงหน้า
- การกำหนดลำดับความสำคัญของคิว (Priority Queue)
- การบันทึกเสียงหรือวิดีโอการทำงาน
- การส่งออกรายงานเป็นไฟล์ Excel/PDF
- Dark/Light mode toggle
- การแปลภาษาอัตโนมัติ
- การซิงค์ข้อมูลกับระบบอื่นๆ นอกเหนือจาก FiveM server
- การแจ้งเตือนอัตโนมัติเมื่อใกล้ครบชั่วโมงทำงาน
- ระบบลาป่วย/ลากิจ
- ระบบอนุมัติการลา
- การตั้งค่าชั่วโมงทำงานแยกตามบุคคล
